# Security Policy

## Scope

This repository contains an educational reverse-engineering challenge based on a self-contained toy program.

The project is designed for local analysis, debugging, and controlled runtime patching of code that is intentionally provided for that purpose.

## Out of Scope

The following topics are outside the scope of this repository:

- Cheating in commercial games
- Bypassing anti-cheat systems
- Circumventing DRM or license checks
- Attacking production systems
- Modifying third-party software without permission
- Developing malware, loaders, injectors, or persistence mechanisms

## Reporting Security Issues

If you identify a security issue in this repository, open a GitHub issue with:

- A clear description of the issue
- Steps to reproduce it
- Expected behavior
- Actual behavior
- Relevant environment details

Please avoid publishing exploit-style instructions for third-party software or unrelated systems.

## Responsible Use

Use this project only in environments where you have permission to analyze and modify the software.

This repository is intended for education, defensive learning, and reverse-engineering practice in a controlled lab setting.
